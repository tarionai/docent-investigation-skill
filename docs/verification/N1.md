## What was built
- `src/docent_investigation/types.py` — `OracleLabel` enum (RESOLVED / UNRESOLVED / EMPTY_PATCH, `.is_resolved`).
- `src/docent_investigation/transform.py` — pure transform:
  - `openhands_record_to_agent_run(record, oracle=None) -> AgentRun` (one Transcript; per-step tool calls preserved).
  - `load_oracle(summary_path) -> dict[instance_id, OracleLabel]`.
  - Event mapping: `SystemPromptEvent`→system, `MessageEvent`(user/agent)→user/assistant,
    `ActionEvent`→assistant + `ToolCall(id, function=tool_name, arguments=action)`, `ObservationEvent`→`ToolMessage`,
    `ConversationStateUpdateEvent`→dropped. Content blocks flattened (str | dict | list of `{type:text,text}`).
- `scripts/fetch_traces.py` — streams the HF dataset's `output.jsonl` + downloads the oracle summary; gitignored `data/`.
- `tests/fixtures/openhands_record.json` (127KB real record, astropy__astropy-13033) + `oracle_summary.json` (45KB).

## Honesty design decision (refines plan §N1)
Oracle is stored in `AgentRun.metadata.scores.resolved` for the public-collection UI, but the rubric eval will
run with `include_metadata=False` (N2) so the judge stays BLIND to the oracle. Transform is oracle-agnostic by
default (`oracle=None` ⇒ no scores) so it can never accidentally leak. Verified by `test_oracle_is_stored_only_when_provided`.
