## Bug caught by the cold clone (and fixed)
`plugin_sanity._check_forbidden` used `Path.rglob("*")`, which walked into `.venv` and crashed on a
dangling long path inside an installed dependency (`inspect_ai/.../__snapshots__`). Fixed: switched to
`os.walk` with a `SKIP_DIRS` prune set (.venv/.git/data/__pycache__/build/…), so only the plugin's own
files are scanned. Also excluded `.venv` from the test's `copytree`. Sanity now runs in ~1s and is robust
to an in-repo venv. This is exactly what the cold-clone gate is for.
